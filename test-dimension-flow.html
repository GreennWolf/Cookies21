🔍 Test de Flujo de Dimensiones - Debug Console

📋 Instrucciones para depurar el problema de dimensiones

Paso 1: Abre la aplicación del banner editor en otra pestaña con el puerto 5173
Paso 2: Abre las DevTools (F12) y ve a la pestaña Console
Paso 3: Crea o selecciona un banner y luego:
  • Agrega un contenedor
  • Agrega una imagen dentro del contenedor
  • Selecciona la imagen dentro del contenedor
Paso 4: En el panel de propiedades, intenta cambiar las dimensiones:
  • Cambia el width de la imagen (ej: 50px o 80%)
  • Cambia el height de la imagen (ej: 60px o 90%)
Paso 5: Observa los logs en la consola y busca:
  • 🎨 BannerPropertyPanel - handleStyleChange
  • 🎯 BannerEditor - handleUpdateStyle
  • 🎨 ComponentRenderer
Paso 6: Verifica si:
  • Los valores llegan correctamente a handleStyleChange
  • Se detecta correctamente si es componente hijo
  • Los estilos se aplican en ComponentRenderer
  • Los cambios se reflejan visualmente

🐛 Síntomas del problema reportado:

  • Containers: Los cambios en dimensiones no tienen efecto
  • Imágenes dentro de containers: Los cambios en dimensiones no tienen efecto
  • Sistema de porcentajes: No funciona correctamente
  • Contexto: Los porcentajes deberían usar el canvas como referencia para componentes directos, y el contenedor padre para componentes hijos

📊 Puntos de control en el flujo:

  1. Input del usuario: Cambio en DimensionControl
  2. BannerPropertyPanel.handleStyleChange: Recibe el cambio
  3. BannerEditor.handleUpdateStyle: Procesa la actualización
  4. useBannerEditor.updateChildStyleForDevice/updateComponentStyleForDevice: Actualiza el estado
  5. ComponentRenderer: Aplica los estilos al DOM
  6. Efecto visual: Se debe ver el cambio en el canvas

🔧 Posibles causas del problema:

  • Los valores no llegan correctamente desde el input
  • La detección de componente hijo/padre es incorrecta
  • Los estilos se pierden durante el procesamiento
  • El ComponentRenderer no está aplicando correctamente los estilos
  • Hay conflictos de CSS que anulan los cambios
  • El sistema de porcentajes está convirtiendo incorrectamente

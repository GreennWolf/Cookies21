Lógica corregida para orden visual invertido

🎯 Problema Identificado

❌ Al invertir la lista visual, se rompió la comunicación de índices entre LayersPanel y BannerEditor

Flujo de Datos Corregido:

LayersPanel

Lista visual invertida

Usa realIndex

Drag & Drop

Envía índices reales

No índices visuales

BannerEditor

Recibe índices reales

Reordena array original

⚙️ Correcciones Implementadas

1. LayersPanel - Mapeo Correcto:

// Lista visual invertida pero índices reales {[...components].reverse().map((component, reverseIndex) => { const realIndex = components.length - 1 - reverseIndex; return ( <LayerItem zIndexNumber={realIndex + 1 + 100} arrayIndex={realIndex} // ¡ÍNDICE REAL! /> ); })}

2. Drag & Drop - Índices Reales:

// handleDrop usa índices reales del array original const sourceRealIndex = dragData.sourceIndex; // Índice real const targetRealIndex = arrayIndex; // Índice real del target onReorder(sourceRealIndex, targetRealIndex); // ¡REALES!

3. BannerEditor - Validación Robusta:

// Validar y ajustar índices para evitar errores if (sourceIndex < 0 || sourceIndex >= components.length) { console.error('❌ Source index inválido'); return; } const adjustedDestinationIndex = Math.max(0, Math.min(destinationIndex, components.length - 1)); if (sourceIndex === adjustedDestinationIndex) { console.log('⚠️ Índices iguales, no se reordena'); return; }

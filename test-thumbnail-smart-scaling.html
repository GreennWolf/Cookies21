🎯 Smart Thumbnail Scaling Implementation

📏 Scaling Rules

🟢 Porcentajes: SIN escalado (scale = 1.0)

Regla: Si las dimensiones del componente están en porcentajes (%), no aplicar escalado

Razón: Los porcentajes se adaptan automáticamente al contenedor padre

🔵 Píxeles: CON escalado (scale = 0.7)

Regla: Si las dimensiones del componente están en píxeles (px), aplicar escalado 0.7

Razón: Los píxeles son fijos y necesitan reducirse para el thumbnail

💻 Implementación Técnica

Función de Detección:

const needsScaling = () => { const style = childDeviceStyle; // Verificar si alguna dimensión está en píxeles const hasPixelDimensions = (style.width && style.width.includes('px')) || (style.height && style.height.includes('px')) || (style.minWidth && style.minWidth.includes('px')) || (style.minHeight && style.minHeight.includes('px')) || (style.maxWidth && style.maxWidth.includes('px')) || (style.maxHeight && style.maxHeight.includes('px')); return hasPixelDimensions; }; const scaleValue = needsScaling() ? 0.7 : 1;

Aplicación del Escalado:

// Para hijos del contenedor childStyles = { ...childDeviceStyle, position: 'relative', top: 'auto', left: 'auto', right: 'auto', bottom: 'auto', transform: `scale(${scaleValue})`, // 0.7 para px, 1.0 para % transformOrigin: 'center' }; // Para el contenedor principal containerStyles = { ...baseStyles, transform: `scale(${containerScaleValue})`, // 0.7 para px, 1.0 para % transformOrigin: 'top left' };

🔍 Casos de Uso

Caso 1: Banner Responsive (Porcentajes)

// Componente configurado con porcentajes { type: 'container', style: { desktop: { width: '100%', // ← Porcentaje height: 'auto', // ← Auto padding: '2%' // ← Porcentaje } }, children: [ { type: 'button', style: { desktop: { width: '30%', // ← Porcentaje height: '100%' // ← Porcentaje } } } ] } // Resultado: NO scaling (scale = 1.0) // Los porcentajes se adaptan naturalmente al thumbnail

Caso 2: Banner Fixed (Píxeles)

// Componente configurado con píxeles { type: 'container', style: { desktop: { width: '400px', // ← Píxeles height: '200px', // ← Píxeles padding: '20px' // ← Píxeles } }, children: [ { type: 'button', style: { desktop: { width: '120px', // ← Píxeles height: '40px' // ← Píxeles } } } ] } // Resultado: SÍ scaling (scale = 0.7) // 400px → 280px, 200px → 140px, etc.

✅ Beneficios de Este Enfoque

  • 🎯 Inteligente: Respeta la intención del diseño (responsive vs fixed)
  • 📱 Responsive: Los componentes con porcentajes mantienen proporciones
  • 📏 Escalable: Los componentes con píxeles se reducen apropiadamente
  • 🔄 Automático: No requiere configuración manual del usuario
  • 🎨 Consistente: Mantiene la relación visual entre componentes

🧪 Testing Checklist

Para Componentes con Porcentajes:

  • [ ] Width/height en % → Sin escalado aplicado
  • [ ] Mantienen proporciones dentro del thumbnail
  • [ ] Se adaptan al tamaño del contenedor padre

Para Componentes con Píxeles:

  • [ ] Width/height en px → Escalado 0.7 aplicado
  • [ ] Tamaño reducido pero legible en thumbnail
  • [ ] Mantienen proporciones entre elementos

Para Componentes Mixtos:

  • [ ] Si CUALQUIER dimensión está en px → Escalado 0.7
  • [ ] Solo si TODAS las dimensiones son % → Sin escalado

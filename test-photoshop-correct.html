🎨 Lógica Photoshop Corregida

Arriba en la lista = Mayor z-index = Más visible

🔧 Problema Identificado y Corregido

Error: El componente que estaba abajo en la lista tenía mayor z-index.
Corrección: Ahora el componente arriba en la lista tiene mayor z-index (estilo Photoshop).

❌ ANTES (Incorrecto)

Lista en orden normal, z-index invertido

pos:0 Texto Principal 103
pos:1 Botón Aceptar 102
pos:2 Logo 101

❌ El Logo (abajo) se veía menos que el Texto (arriba)

✅ AHORA (Correcto)

Lista invertida, z-index correcto

pos:0 Logo 103
pos:1 Botón Aceptar 102
pos:2 Texto Principal 101

✅ El Logo (arriba) se ve más que el Texto (abajo)

🎯 Demostración Visual

Los componentes están superpuestos. El que está arriba en la lista debe verse por encima:

Logo (z:103)
Botón (z:102)
Texto (z:101)

⚙️ Cambios Implementados

1. Lista Invertida en LayersPanel:

{[...components].reverse().map((component, reverseIndex) => { const realIndex = components.length - 1 - reverseIndex; return ( <LayerItem zIndexNumber={realIndex + 1 + 100} // Arriba = mayor z-index arrayIndex={realIndex} // Índice real del array /> ); })}

2. Z-Index Correcto en BannerEditor:

// Photoshop style: index 0 = mayor z-index components.forEach((comp, index) => { const zIndex = components.length - index + 100; comp.style[device].zIndex = zIndex; console.log(`${comp.id} = ${zIndex} (pos:${index}) - ${index === 0 ? 'FRENTE' : 'atrás'}`); });

3. Mapeo Correcto:

Array real: [Logo, Botón, Texto] // índices: 0, 1, 2 Z-index: [103, 102, 101] // mayor a menor Lista visual: Logo (arriba) → Texto (abajo) Resultado: Logo se ve por encima ✅

🚀 ¡Lógica Photoshop Implementada!

Ahora funciona exactamente como Photoshop:
• Arrastra un componente hacia arriba en la lista = Se mueve al frente (mayor z-index)
• Arrastra un componente hacia abajo en la lista = Se mueve hacia atrás (menor z-index)
